## Overview
Cryptolock spins up a **real Splunk** pre-loaded with the telemetry of a ransomware outbreak on Northwind's fleet. A binary detonates on an endpoint, deletes the shadow copies, inhibits Windows recovery, stops the AV, mass-encrypts files (`.cryptolock`) and drops ransom notes. You reconstruct the whole outbreak with real **SPL** and recover the operator's flag.

## Scenario
Sysmon process and file-creation events are in Splunk. Somewhere in a normal day, a ransomware detonated. Find the ransomware process and patient zero, prove the shadow-copy deletion and recovery inhibition, count the encrypted files, name the ransom note — and decode the obfuscated command for the flag.

## Architecture
Two containers: **splunk** (Splunk Enterprise, HEC, index `northwind`) and **forge** (a deterministic data-forge that generates the ransomware telemetry and ingests it via HEC).

```
forge ──HEC──▶ Splunk (index=northwind)  ◀── you (Splunk Web :8000, real SPL)
```

## Learning Objectives
- Investigate a ransomware outbreak in a real SIEM with SPL.
- Detect defense evasion: shadow-copy deletion, recovery inhibition, AV kill.
- Quantify the impact (mass file encryption) and find patient zero.
- Recognise ransom notes and the ransomware process.
- Turn findings into detections (SPL / Sigma).

## Compatibility — amd64 only
This lab is **amd64 (x86_64) only**: the official **Splunk container image is not published for ARM**. Intel/AMD runs natively; Apple Silicon / ARM runs it **under emulation** (slower, more RAM — give Docker ≥ 6 GB). `./doctor.sh` warns you.

## Deployment
`./deploy.sh` checks the environment, pulls Splunk (Docker Hub) and the `forge` image (GHCR), brings both up, waits until Splunk is healthy and the dataset is ingested, and prints the Splunk URL + credentials.

> Maintainers: `docker compose -f docker-compose.yml -f docker-compose.build.yml build` (local builds use a decoy flag; the real one is injected only in CI).

## Evidence
All evidence is in Splunk, `index=northwind sourcetype=Sysmon`: EventID 1 (process creation) and EventID 11 (file creation — encrypted files and ransom notes).

## Investigation Flow
1. `./deploy.sh`, then `./verify.sh`. Open Splunk Web (`:8000`).
2. Find the mass file encryption (EventID 11, `*.cryptolock`) and count it.
3. Prove the defense evasion: shadow-copy deletion (`vssadmin`), recovery inhibition (`bcdedit`), AV kill.
4. Identify the ransomware process and patient zero; name the ransom note.
5. Decode the `-EncodedCommand` for the flag.
6. **Answer the questions inside KrakenSOC** to earn XP.

Starter searches:
```spl
index=northwind sourcetype=Sysmon EventID=11 TargetFilename="*.cryptolock" | stats count by Computer
index=northwind sourcetype=Sysmon EventID=1 CommandLine="*delete shadows*"
index=northwind sourcetype=Sysmon EventID=1 Image="*cryptolock.exe"
```

## Troubleshooting

## FAQ
**Does Docker grade my answers?** No. Splunk is the environment; questions, flags and scoring live in KrakenSOC.
**Where's the solution?** Encrypted in `answers/` and unlocked from KrakenSOC (SFRS §8).
**Is the data real?** No — synthetic and deterministic, generated by `runtime/forge/`. No real files are encrypted.
