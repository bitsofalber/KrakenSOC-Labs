## Overview
Pilfer spins up a **real Splunk** pre-loaded with the telemetry of an **insider** stealing data. A departing employee plugs in a USB drive, reads sensitive files (payroll, customer lists, contracts), stages them, and copies them to the removable drive. No malware — the threat is internal. You reconstruct it with real **SPL** and recover the flag.

## Scenario
Windows Security and Sysmon events are in Splunk. An employee on their way out took company data with them. Find who did it, what USB drive they used, how many files they copied, the most sensitive file, and the host — then decode the obfuscated command for the flag.

## Architecture
Two containers: **splunk** (Splunk Enterprise, HEC, index `northwind`) and **forge** (a deterministic data-forge that generates the insider telemetry and ingests it via HEC).

```
forge ──HEC──▶ Splunk (index=northwind)  ◀── you (Splunk Web :8000, real SPL)
```

## Learning Objectives
- Investigate insider data theft in a real SIEM with SPL.
- Correlate USB device events, sensitive file access and file copies to a removable drive.
- Recognise local data staging (compression) before exfiltration.
- Quantify the theft and identify the most sensitive asset lost.
- Turn findings into detections (SPL / Sigma).

## Compatibility — amd64 only
This lab is **amd64 (x86_64) only**: the official **Splunk container image is not published for ARM**. Intel/AMD runs natively; Apple Silicon / ARM runs it **under emulation** (slower, more RAM — give Docker ≥ 6 GB). `./doctor.sh` warns you.

## Evidence
All evidence is in Splunk, `index=northwind`: `WinEventLog:Security` (6416 external device, 4663 file access) and `Sysmon` (EventID 1 process, EventID 11 file create on the USB drive).

## Investigation Flow
1. `./deploy.sh`, then `./verify.sh`. Open Splunk Web (`:8000`).
2. Find the USB device recognition (6416) — who and which host.
3. Follow the sensitive file access (4663) and the copies to `E:\` (Sysmon 11).
4. Count the files, name the most sensitive one, spot the staging (Compress-Archive).
5. Decode the `-EncodedCommand` for the flag.
6. **Answer the questions inside KrakenSOC** to earn XP.

Starter searches:
```spl
index=northwind sourcetype=WinEventLog:Security EventCode=6416 Device_Description="*USB*"
index=northwind sourcetype=Sysmon EventID=11 TargetFilename="E:\\*"
index=northwind sourcetype=WinEventLog:Security EventCode=4663
```

## FAQ
**Does Docker grade my answers?** No. Splunk is the environment; questions, flags and scoring live in KrakenSOC.
**Where's the solution?** Encrypted in `answers/` and unlocked from KrakenSOC (SFRS §8).
**Is the data real?** No — synthetic and deterministic, generated by `runtime/forge/`. No real PII.
